# visualisator_project 
# ВИЗУАЛИЗАТОР ПРОДАЖ

## ОПИСАНИЕ
Консольное приложение для анализа данных о продажах с визуализацией результатов в виде графиков. Программа предоставляет интерактивное меню для выбора различных типов анализа и отображает соответствующие графики в отдельных окнах.

## ФУНКЦИОНАЛЬНЫЕ ВОЗМОЖНОСТИ

### Анализ выручки по времени
- **Выручка по дням** - линейный график ежедневной выручки
- **Выручка по неделям** - линейный график с группировкой по неделям
- **Выручка по месяцам** - столбчатая диаграмма месячной выручки

### Анализ категорий товаров
- **Распределение выручки по категориям** - круговая диаграмма
- **Топ-5 подкатегорий по выручке** - горизонтальная столбчатая диаграмма
- **Топ-5 подкатегорий по прибыли** - горизонтальная столбчатая диаграмма

### Географический анализ
- **Топ-5 штатов по выручке** - горизонтальная столбчатая диаграмма
- **Топ-5 городов по выручке** - горизонтальная столбчатая диаграмма

### Динамика продаж за период
- **Произвольный временной интервал** - выбор начальной и конечной даты
- **Несколько метрик** - выручка, количество заказов или прибыль
- **Различные периоды агрегации** - дни, недели, месяцы, кварталы
- **Адаптивный тип графика** - линейные графики для коротких периодов, столбчатые для длинных

### Анализ клиентов
- **Топ-5 клиентов по сумме заказов** - столбчатая диаграмма
- **Детальная статистика клиентов** - количество заказов, средний чек, общая прибыль

## АРХИТЕКТУРА ПРОЕКТА

### Структура файлов
```
├── main.py                    # Точка входа в программу
├── config.py                  # Конфигурационные параметры
├── data_loader.py            # Загрузка и предобработка данных
├── interface.py              # Пользовательский интерфейс
├── utils.py                  # Вспомогательные функции
├── data/                     # Папка с исходными данными
│   └── Sales_data.csv       # Данные о продажах в формате CSV
└── modules/                  # Модули анализа
    ├── __init__.py          # Инициализация пакета модулей
    ├── revenue_time.py      # Анализ выручки по времени
    ├── category_analysis.py # Анализ категорий товаров
    ├── geo_analysis.py      # Географический анализ
    ├── dynamic_sales.py     # Анализ динамики продаж
    └── customer_analysis.py # Анализ клиентов
```

### Модульная организация
- **main.py** - координирует запуск программы и обработку ошибок
- **data_loader.py** - отвечает за чтение CSV файла и преобразование данных
- **interface.py** - реализует консольное меню и навигацию
- **utils.py** - содержит общие утилиты для работы с графиками
- **modules/** - каждый модуль отвечает за конкретный тип анализа

### Поток данных
1. Загрузка данных из CSV файла
2. Преобразование и обогащение данных
3. Выбор типа анализа через консольное меню
4. Обработка данных в соответствующем модуле
5. Создание и отображение графика
6. Возврат к меню после закрытия окна графика

## ОСОБЕННОСТИ РАБОТЫ

### Входные данные
- Формат файла: CSV с разделителями-запятыми
- Обязательные поля: Order Date, Amount, Profit, Category, Sub-Category, CustomerName, State, City
- Поддерживаемый формат дат: ГГГГ-ММ-ДД

### Режим отображения графиков
- Все графики отображаются в интерактивных окнах
- Программа приостанавливает выполнение до закрытия окна графика
- Отсутствие автоматического сохранения в файлы
- Возможность сохранения графика через стандартные средства окна просмотра

### Пользовательский интерфейс
- Иерархическая система меню
- Обработка некорректного ввода
- Информативные сообщения об ошибках
- Статистика данных при загрузке

### Особенности анализа
- **Динамический анализ** - поддержка произвольных временных интервалов
- **Адаптивная визуализация** - выбор типа графика в зависимости от периода
- **Локализованный формат валюты** - автоматическое форматирование сумм (K для тысяч, M для миллионов)
- **Конфигурируемый топ-N** - централизованное управление количеством элементов в топах

## ТРЕБОВАНИЯ К СИСТЕМЕ

- Python 3.7 или выше
- Библиотеки: pandas, matplotlib

### Структура данных
Файл `Sales_data.csv` должен находиться в папке `data/` относительно корня проекта.

## ПРИНЦИПЫ РАБОТЫ

### Обработка данных
- Автоматическое преобразование дат из строкового формата
- Создание дополнительных временных измерений (год, месяц, неделя, квартал)
- Группировка и агрегация данных в соответствии с выбранным анализом

### Визуализация
- Использование темы seaborn-whitegrid для всех графиков
- Единая цветовая схема в соответствии с config.py
- Автоматическое масштабирование графиков
- Читабельные подписи осей и значений

### Управление программой
- Бесконечный цикл основного меню до явного выхода
- Возможность многократного построения графиков без перезапуска
- Сохранение состояния данных между сеансами анализа
